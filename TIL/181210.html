1. 브라우저가 어떻게 구성 되어있느냐?
    Browser Components : 우리가 사용하는 인터넷 그 자체의 영역
    User Interface : chrome위에 새 탭, 이라던지, 뒤로가기, 닫기, 최소화 이런것들이 UI 이다. OS레벨에서 설치된 그런 것들임.
    browserEngine : 소스코드를 실행해 주는 엔진. browser소프트웨어를 동작시켜주는 핵심엔진
    Rendering Engine : 화면을 잡고, 색칠을 해주는, 픽셀단위로 어떤것들을 그려 눈으로 볼 수 있게 해주는 엔진
    Data persistence : browser또한 데이터를 저장(캐싱)하는 일을 함. 그거를 관리하는 영역
    Networking : 서버와 통신하는 영역(http를 통해서)
    javascript interpreter : 자바스크립트를 해석해주는 영역
    UI Backend : UI영역을 처리시켜주는 백엔드 영역
    url :https://www.html5rocks.com/en/tutorials/internals/howbrowserswork/#The_browser_high_level_structure
  2. rendering 영역
    브라우저별로 다른 것을 가지고 있음.
    Mainflow
    1. Parsing HTML to consturct the DOM tree : 구조화된 HTML 데이터를 해석해서 DOM, 일종의 트리 형태로 데이터를 다시 가지고 있게 됨
    2. Render tree construction : 바로 그게 렌더트리를 만듬
    3. Layout of the render tree : 렌더트리를 기준으로 css를 합쳐서 만듬.
    4. Painting the render tree : 렌더트리를그려줌
  3. https://www.html5rocks.com/en/tutorials/internals/howbrowserswork/#The_main_flow html파일로 돔트리를 만들고, css 파일로 돔트리의 스타일을 만들어줌. 그리고 그 두가지를 합쳐서 렌더트리가 그려지고, 그다음 레이아웃이 결정되고(어디에, 크기는어느정도?) 화면에 페인팅 작업이 이루어 진 다음 display
  4. parsing-general
    2 + 3 - 1의 내용을 파싱하려면, 2, +. 3, -, 1 이렇게 각자 따로따로 나누고 이것들이 가진 의미로 연산을 실행한다. 토큰을 기준으로 syntax tree를 만들고 그로 인해 실행되게 됨. 그냥 일종의 컴파일러 부분에 해당. 토큰단위로 잘라서 의미를 해석하고 그 의미에 따라 실행되는것.
  5. html parser
    html의 기본적인 것으로 짜여진 것을 마크업이라고 함. 이것들을 브라우저가 저장할때 아래 url의 그림과 같은 모습으로 저장이 됨.
  6. css parsing
    css file 또한 html파일과 다를바 없이 토큰화되어 데이터를 읽어서 트리구조로 실행이 된다.
  7. html태그들.
    p> : Paragraph 의 p 문단을 작성하는데 사용.
    h1>~h6> : 문단의 제목에 사용. 요즘엔 중요도를 우선시한다고 함.
    hr> : 단락구분, 문서의 구분 선(닫힘 없음)
    br> : break의 약자. 줄바꿈을 행할때 사용(닫힘 없음)
    div> : division의 약자. HTML문서 영역의 구분이나 섹션의 분할을 의미함.
    span> : inline요소. 줄바꿈은 되지 않는다. 폭 높이 요소 또한 적용받지 아니한다. inline요소들을 묶는데 사용되며 브라우저의 레이아웃은 변경되지 않음.
    table> : 표.  border, cellpadding, cellspacing, width, summary의 속성값이 따로 있으며. 세로정렬이 자동으로 된다. 보통 게시판에 많이 이용함
    captain> : summary역할, css에서 안보이게 해준다.
    colgroup> : 열 그룹, 그룹화를 위해서 사용된다.
    col..> : 빈 태그중에 유일하게 self close태그가 없음. (닫힘 없음)
    thead>, tbody>, tfoot> : 위서부터 차례대로 테이블 헤더 행 그룹, 테이블 내용 행 그룹, 테이블 바닥 행 그룹, 계산시 합계 부분에 자주 이용.
    ol> : ordered list 순서가 있다. 앞에 숫자가 붙음
    ul> : unordered list 순서가 없다. 앞에 기호가 붙음.
    li> : ol과 ul의 세부항복, ol과 ul의 자식으로는 li만 올 수 있다
    a> : anchor의 약자. a태긔의 속성
    href>-목적지, url,
    target>-보여질 위치
    *위 타켓의 속성.
    _blank>-새창
    _self>-이동
    _parent>-먼저 띄워진 창
    _top>- 현재 띄워진 가장 최상단에 위치한 창.
    name>- a tag의 이름 지정.
    title>- 링크에 대한 설명
    img> : 이미지 가져오기, inline요소
    src>- 경로지정(필수입력)
    alt>-이미지를 위한 대체 텍스트
    title>-별로 사용하지 않음
    input> : 입력상자. 속성에  type, name, value, checked, maxlength, readonly, disabled
    center> : 내의 값에 들어가는 모든 객체값은 중앙으로 정렬된다.
    strong> : 글씨를 굵게 표현한다.
    dl>, dt>, dd> : https://developer.mozilla.org/en-US/docs/Web/HTML/Element/dl
    article>, section>, div> : article은 내용이 독립적이고 스스로 설 수 있는 내용이라면 사용
    section은 내용이 서로 관계가 있다면 사용
    div의미적으로 관계가 없다면 사용.
